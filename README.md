# Travel Reservation Application – SRS

This repository contains a Software Requirements Specification (SRS) document for a travel reservation application called "WayToGo".


## Project Overview

The system is designed to provide users with a seamless and convenient platform to book and manage travel services, including flights, hotels, car rentals, and tourist activities.

The application focuses on delivering a smooth user experience with a structured and user-friendly interface.


## System Features

The system supports multiple travel-related services, including:

- Flight booking
- Hotel reservation
- Car rental
- Booking tourist attractions and events
- Hiring a tour guide
- Travel companion selection
- Reservation tracking and management


## Functional Requirements

Key functionalities include:

- User authentication (Sign up / Login / Absher integration)
- Search across travel categories (hotels, flights, attractions, etc.)
- Booking and reservation management
- Profile and settings management
- Payment processing and confirmation
- Error handling for invalid inputs and system failures


## Non-Functional Requirements

The system ensures:

- High performance and responsiveness
- Data security (encryption and secure protocols)
- Reliability and availability
- Maintainability and scalability
- User accessibility and usability


## Documentation Contents

The SRS document includes:

- Introduction and system scope
- Overall system description
- Functional and non-functional requirements
- External interface requirements
- Use case diagrams
- System workflows and UI design
